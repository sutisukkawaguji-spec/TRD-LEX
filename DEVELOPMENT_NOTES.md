# คู่มือการพัฒนาโครงการ TRD Lease Exchange (TRD-LEX)

เอกสารฉบับนี้จัดทำขึ้นเพื่อบันทึกโครงสร้างทางเทคนิค การตั้งค่าระบบ และจุดสำคัญของโค้ด เพื่อให้ผู้พัฒนาหรือ AI ในรอบการทำงานถัดไปสามารถอ่าน เข้าใจ และพัฒนาต่อยอดระบบได้ทันที

---

## 1. ภาพรวมระบบ (System Overview)
**TRD Lease Exchange (TRD-LEX)** เป็นระบบตลาดรองสำหรับการโอนสิทธิการเช่าที่ราชพัสดุ ดำเนินการภายใต้ข้อกำหนดและพระราชบัญญัติที่ราชพัสดุ พ.ศ. 2562 ของกรมธนารักษ์
* **ความสำคัญ:** ไม่ใช่การซื้อขายที่ดิน แต่เป็นระบบตรวจสอบสิทธิ์และอำนวยความสะดวกในการ **"โอนเปลี่ยนชื่อผู้เช่าในสัญญาเช่าราชพัสดุ"** อย่างถูกต้องตามระเบียบกระทรวงการคลัง

---

## 2. เทคโนโลยีที่ใช้งาน (Technology Stack)
* **Core:** Next.js 16.2 (App Router) + React 19 + JavaScript
* **Styling:** Vanilla CSS (ใช้ระบบ CSS Variables ร่วมกับ `globals.css`)
* **Map:** Leaflet Map (แสดงพิกัดขอบเขตที่ดินและวาดเส้นทางจากแปลงที่ดินไปยังสถานี BTS/MRT)
* **Authentication:** Mocked Digital ID (ThaiD - กรมการปกครอง)

---

## 3. สถาปัตยกรรมและโครงสร้างไฟล์หลัก (Key File Structure)
ตัวโปรเจกต์ Next.js ทั้งหมดจะติดตั้งอยู่ภายในโฟลเดอร์ย่อย `/trd-lex` โดยมีโครงสร้างที่สำคัญดังนี้:

* **`/trd-lex/lib/utils.js` [ใหม่]:**
  * มีฟังก์ชันสำคัญคือ `getAssetUrl(path)` ใช้ควบคุมการแปลง URL ของรูปภาพในโฟลเดอร์ `/public` ทั้งหมดให้ทำงานสอดคล้องกับ `basePath` บน GitHub Pages เมื่ออยู่บนโปรดักชัน แต่จะไม่รบกวนการทำงานของ Blob URL (`blob:`) ตอนผู้ใช้อัปโหลดรูปภาพจำลอง
* **`/trd-lex/components/ListingDetailClient.js` [ใหม่]:**
  * คอมโพเนนต์ฝั่ง Client ที่รวบรวมฟังก์ชันการโต้ตอบทั้งหมดของหน้ารายละเอียดประกาศ (สไลเดอร์, การขยายแผนที่เต็มจอ, ปุ่มนำเส้นทางไป BTS, และกล่องแสดงสาธารณูปโภคแบบ Avatar)
* **`/trd-lex/app/listings/[id]/page.js`:**
  * ปรับปรุงเป็น Server Component ทำหน้าที่เรียกฟังก์ชัน `generateStaticParams()` เพื่อให้ Next.js บิลด์หน้าแบบ Static HTML ออกมาได้ผ่านฉลุย
* **`/trd-lex/app/sell/page.js`:**
  * ฟอร์มลงประกาศ 4 ขั้นตอน: 1. ยืนยันตัวตน ThaiD -> 2. ดึงสัญญาจริงของผู้ถือครองจากระบบกรมธนารักษ์พร้อมขยายดูรายละเอียดเชิงลึก -> 3. กรอกข้อมูลเฉพาะด้านพร้อมจุดอัปโหลดภาพถ่ายจริงที่ถอดถอนพรีวิวได้ และแผนที่ Leaflet ให้ติ๊กพิกัดที่ดิน -> 4. แสดงการคำนวณอัตราประมาณการโอนสิทธิ์และเตือนกฎหมาย พ.ร.บ. ที่ราชพัสดุ 2562
* **`/trd-lex/app/login/page.js`:**
  * หน้าจอลงทะเบียนสมัครสมาชิกที่ควบคุมไม่ให้กดสมัครจนกว่าจะกดยอมรับระเบียบกฎหมายของกรมธนารักษ์
* **`/trd-lex/components/NewsletterSection.js`:**
  * บล็อกสมัครติดตามประกาศโอนสิทธิ์แปลงที่ดินออกใหม่บริเวณด้านล่างของหน้าแรก

## 4. ข้อกำหนดการคอมไพล์และการดีพลอย (Build & Deployment)

### 4.1 การพัฒนาและทดสอบในเครื่องคอมพิวเตอร์ (Local Development)
เนื่องจากชื่อโฟลเดอร์โครงการเป็นภาษาไทย (`OneDrive/เอกสาร/GitHub/...`) ทำให้ตัวคอมไพเลอร์ใหม่อย่าง Turbopack เกิดปัญหาการเข้ารหัสชื่อไดเรกทอรีบน Windows (Unicode Path Panics)
> ⚠️ **กฎเหล็กการรันในเครื่อง:**
> * **รันโหมดพัฒนา:** ให้รันผ่าน Webpack เสมอโดยพิมพ์:
>   ```bash
>   npm run dev
>   ```
>   *(สคริปต์นี้ถูกกำหนดให้ใช้คำสั่ง `next dev --webpack` ไว้เรียบร้อยแล้วใน `package.json`)*
> * **สั่งบิลด์ทดสอบในเครื่อง:** ให้ใช้ Webpack เช่นกัน:
>   ```bash
>   npx next build --webpack
>   ```

### 4.2 การดีพลอยไปยัง GitHub Pages (GitHub Pages Deployment)
ระบบถูกกำหนดค่าให้ทำงานร่วมกับ GitHub Actions บนคลาวด์ได้อย่างไร้ข้อผิดพลาด:
* **ไฟล์ตั้งค่าบิลด์ (`/trd-lex/next.config.mjs`):**
  * `output: 'export'` (แปลงระบบเป็น HTML ส่งออก Static)
  * `trailingSlash: true` (สร้างโฟลเดอร์สำหรับแต่ละหน้าแก้ปัญหา 404 เมื่อกดรีเฟรชหน้าเว็บ)
  * `basePath: '/TRD-LEX'` (เพิ่ม Prefix ให้สอดคล้องกับ URL ของคลังบน GitHub)
  * `images: { unoptimized: true }` (อนุญาตการใช้งานภาพถ่ายบน Static Host)
* **สคริปต์อัตโนมัติ (`/.github/workflows/deploy.yml`):**
  * ควบคุมการ Checkout, ติดตั้งโมดูลด้วย `npm ci`, สั่งบิลด์ `npm run build` ในโฟลเดอร์ย่อย `trd-lex` และนำโฟลเดอร์บิลด์ปลายทาง `/out` ไปดีพลอยเป็นโฮสต์สาธารณะแบบอัตโนมัติเมื่อพุชขึ้นสาขา `main`

## 5. แผนการพัฒนาต่อยอดที่แนะนำ (Next Steps)
หากมีรอบการพัฒนาถัดไป หัวข้อต่อไปนี้เป็นส่วนสำคัญที่แนะนำให้ทำ:
1. **การเชื่อมต่อ API สารบบที่ราชพัสดุจริง:** เปลี่ยนการ Mock สัญญาเช่าของ นายสมชาย ทดสอบ ในขั้นตอนลงประกาศโอนสิทธิ์ให้เป็นการเชื่อมฐานข้อมูล Database จริง
2. **ระบบการจองทำเลและชำระค่าธรรมเนียมเบื้องต้น:** เพิ่มระบบตะกร้าสินค้าหรือการมัดจำเพื่อล็อกสิทธิ์การเข้าเจรจาโอนสิทธิ์สัญญา
3. **การกรองค้นหาที่ดินขั้นสูง:** เพิ่มระดับการกรองแปลงที่ดินตามระยะห่างสถานีรถไฟฟ้า (เช่น ระยะห่างภายใน 500 เมตร, 1 กิโลเมตร)
